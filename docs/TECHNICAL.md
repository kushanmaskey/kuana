## 5. Authentication & Authorization  <!-- was §4 -->

### Method
- **JWT (JSON Web Token)** signed with `JWT_SECRET`
- Token lifetime: **8 hours**
- Stored client-side in `localStorage` under key `kuana_token`
- Sent on every authenticated request as `Authorization: Bearer <token>`

### Middleware — `server/middleware/auth.js`

```js
function requireAuth(req, res, next) {
  const authHeader = req.headers.authorization;
  if (!authHeader || !authHeader.startsWith('Bearer ')) {
    return res.status(401).json({ error: 'Unauthorized' });
  }
  const token = authHeader.split(' ')[1];
  try {
    req.admin = jwt.verify(token, process.env.JWT_SECRET);
    next();
  } catch {
    return res.status(401).json({ error: 'Invalid or expired token' });
  }
}
```

### Token Payload

```json
{
  "id": 1,
  "email": "admin@kuana.org",
  "role": "admin",
  "iat": 1700000000,
  "exp": 1700028800
}
```

### AUTH

#### `POST /api/auth/login`
Login and receive a JWT token.

**Rate limit:** 10 requests per IP per 15 minutes.

**Request body:**
```json
{ "email": "admin@kuana.org", "password": "mypassword" }
```

**Success `200`:**
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "admin": { "id": 1, "name": "Admin", "email": "admin@kuana.org", "role": "admin" }
}
```

**Errors:**
| Code | Reason |
|---|---|
| `400` | Missing email or password |
| `401` | Invalid credentials |
| `429` | Rate limit exceeded |

---

#### `POST /api/auth/setup`
One-time endpoint to create the first admin. Blocked if `DISABLE_SETUP=true`.

**Request body:**
```json
{
  "name": "Admin",
  "email": "admin@kuana.org",
  "password": "securepassword",
  "setupKey": "your-setup-key"
}
```

**Errors:**
| Code | Reason |
|---|---|
| `400` | Missing fields or password < 10 chars |
| `403` | Wrong setup key or disabled |
| `409` | Admin already exists |

### ALUMNI

#### `GET /api/alumni` 🔒
List alumni. Supports query filters.

**Query params:**

|---|---|---|

---

#### `POST /api/alumni/register`
Public self-registration. **Rate limit:** 10 per IP per hour.

**Request body:**
```json
{
  "first_name": "Kushan",
  "last_name": "Maskey",
  "email": "kushan@example.com",
  "phone": "+1-555-0100",
  "graduation_year": 2010,
  "degree": "BE",
  "department": "Computer Engineering",
  "city": "Dallas",
  "state_province": "TX",
  "country": "USA",
  "bio": "Software engineer...",
  "linkedin_url": "https://linkedin.com/in/kushanmaskey"
}
```

**Validation:**
- `first_name`, `last_name`, `email` — required
- `email` — must match regex `/^[^\s@]+@[^\s@]+\.[^\s@]+$/`
- `first_name`, `last_name` — max 100 chars each
- `graduation_year` — must be between 1991 and current year + 1
- `bio` — truncated to 1000 chars automatically

**Response `201`:**
```json
{ "success": true, "alumni": { "id": 42, "first_name": "Kushan", "last_name": "Maskey", "email": "kushan@example.com" } }
```

**Errors:** `409` if email already registered.

```sql
  message           TEXT,
  payment_reference TEXT,
  payment_method    VARCHAR(50),
  status            VARCHAR(20) DEFAULT 'completed',
  donated_at        TIMESTAMPTZ DEFAULT NOW()
);
```

#### `media`
```sql
CREATE TABLE media (
  id            SERIAL PRIMARY KEY,
  event_id      INTEGER REFERENCES events(id),
  title         TEXT NOT NULL,

```

### Key Queries Per Route

| Route | Query | Parameterized |
|---|---|---|
| Login | `SELECT * FROM admins WHERE email = $1` | ✅ |
| Setup | `INSERT INTO admins (name, email, password_hash) VALUES ($1,$2,$3)` | ✅ |
| Get events | `SELECT * FROM events WHERE is_published = true ORDER BY event_date DESC` | ✅ |
| Get event | `SELECT * FROM events WHERE id = $1` | ✅ |
| Alumni search | Dynamic with ILIKE `%$1%` | ✅ |
| Register alumni | `INSERT INTO alumni (...) VALUES (...)` | ✅ |
| Soft delete alumni | `UPDATE alumni SET is_active = false WHERE id = $1` | ✅ |
| Contact submit | `INSERT INTO contact_messages (...)` | ✅ |
| Get contacts | `SELECT * FROM contact_messages ORDER BY created_at DESC` | ✅ |
| Mark read | `UPDATE contact_messages SET is_read = true WHERE id = $1` | ✅ |

> ⚠️ All queries use parameterized statements (`$1`, `$2`, ...). **SQL injection is not possible.**

## 7. Admin (CMS) Tool

**URL:** `/admin`
**Access:** Requires valid JWT stored in `localStorage`

### Tabs

| Tab | What it does |
|---|---|
| **Events** | Create, view, delete reunion events. Toggle featured/published status. |
| **Alumni** | Search and browse registered alumni by name/email. |
| **Messages** | View contact form submissions. Mark as read. |
| **Donations** | View donation records and aggregate stats (total, average, unique donors). |

### Session Management
- Token decoded from `localStorage` on page load via `atob(token.split('.')[1])`
- If `exp` is in the past, token is removed and user is redirected to login
- "Sign out" clears `kuana_token` from localStorage

### API Client (`client/src/api.js`)
All calls go through Axios with the token automatically attached via an interceptor.

## 9. Security Controls

| Control | Implementation | Status |
|---|---|---|
| **SQL Injection** | All queries parameterized with `$1`, `$2` | ✅ Secured |
| **Password hashing** | bcrypt with cost factor 12 | ✅ Secured |
| **Timing attack prevention** | bcrypt runs even when user not found (fake hash) | ✅ Secured |
| **JWT authentication** | `jsonwebtoken`, 8h expiry, verified on every protected route | ✅ Secured |
| **Rate limiting — login** | 10 attempts per IP per 15 min | ✅ |
| **Rate limiting — contact** | 5 per IP per hour | ✅ |
| **Rate limiting — alumni register** | 10 per IP per hour | ✅ |
| **Rate limiting — donations** | 10 per IP per hour | ✅ |
| **CORS** | Locked to `CLIENT_URL` / `kuana.org` in production | ✅ |
| **Security headers** | `helmet` middleware | ✅ |
| **Body size limit** | `express.json({ limit: '50kb' })` | ✅ |
| **Soft deletes** | Alumni are deactivated, not erased | ✅ |
| **Setup endpoint** | Disabled by `DISABLE_SETUP=true` after first admin created | ✅ |
| **XSS** | React escapes output by default; no `dangerouslySetInnerHTML` used | ✅ |
| **SSL in production** | PostgreSQL pool uses SSL when `NODE_ENV=production` | ✅ |
| **Token storage** | `localStorage` (vulnerable to XSS if third-party scripts injected) | ⚠️ Acceptable for admin-only tool |

## 10. Debugging Guide with Examples

### 10a. Login Fails with 401

**Symptom:** `POST /api/auth/login` returns `{ "error": "Invalid credentials" }`

**Check:**
```bash
# Verify admin exists in DB
psql $DATABASE_URL -c "SELECT id, email, role FROM admins;"

# Test bcrypt hash manually (Node REPL)
node -e "const b=require('bcryptjs'); b.compare('mypassword', '<hash_from_db>').then(console.log);"
```

**Common causes:**
- Wrong email (case-sensitive in DB, but login lowercases via JS — verify)
- Password was hashed with a different cost factor
- Admin doesn't exist — run `/api/auth/setup` first

---

### 10b. 429 Too Many Requests

**Symptom:** Rate limiter blocks requests during development.

**Fix:** Restart the server (in-memory rate limit resets) or temporarily raise the `max` in the route file. Do NOT disable in production.

---

### 10c. JWT Token Expired

**Symptom:** Admin page shows login screen unexpectedly.

**Debug:**
```js
// Paste in browser console
const token = localStorage.getItem('kuana_token');
const payload = JSON.parse(atob(token.split('.')[1]));
console.log('Expires:', new Date(payload.exp * 1000));
```

**Fix:** Log in again. Token lasts 8 hours.

---

### 10d. Database Connection Fails

**Symptom:** All API calls return `500 Server error`.

**Check:**
```bash
# Test DB connection directly
psql $DATABASE_URL -c "SELECT NOW();"

# Check env var is set
echo $DATABASE_URL
```

**Common causes:**
- `.env` file missing or `DATABASE_URL` not set
- SSL mismatch (`rejectUnauthorized` vs self-signed cert)
- DB server not running

---

### 10e. Alumni Register Returns 409

**Symptom:** `{ "error": "Email already registered" }` (PostgreSQL error code `23505` — unique violation).

**Example input that triggers it:**
```json
{ "first_name": "Kushan", "last_name": "Maskey", "email": "kushan@example.com" }
```
(when `kushan@example.com` already exists in `alumni` table)

**Fix:** Use a different email, or check the DB:
```sql
SELECT id, first_name, last_name, email FROM alumni WHERE email = 'kushan@example.com';
```

---

### 10f. Media Images 404 in Production

**Symptom:** Gallery images from `/assets/img/gallery/2025/` don't load after deployment.

**Cause:** Local images in `client/public/assets/img/gallery/` must be committed to git and included in the Render build.

**Fix:**
```bash
git add client/public/assets/img/gallery/
git commit -m "Add 2025 gallery images"
git push
```

---

### 11g. CORS Error in Development

**Symptom:** `Access to XMLHttpRequest blocked by CORS policy`

**Fix:** Ensure Vite proxy is configured in `client/vite.config.js`:
```js
server: {
  proxy: {
    '/api': 'http://localhost:4000'
  }
}
```
And that the backend is running on port 4000. If you changed `PORT` in `.env`, update the proxy target to match.

---

### 11h. Contact Form Email Not Arriving

**Symptom:** Form submits successfully (200 response) but no email arrives.

**Check server logs** for the error line:
```
Email send failed: Invalid login: 535-5.7.8 Username and Password not accepted
```

**Common causes and fixes:**

| Cause | Fix |
|---|---|
| `MAIL_USER` / `MAIL_PASS` not set in `.env` | Add the values and restart server |
| Using account password instead of App Password | Generate an App Password at myaccount.google.com/apppasswords |
| 2FA not enabled on Google account | Enable 2-Step Verification first, then generate App Password |
| Server not restarted after `.env` change | Restart `node server.js` |

**Test email directly (without the web form):**
```bash
cd server
node -e "
require('dotenv').config();
const { sendContactEmail } = require('./utils/mailer');
sendContactEmail({ name: 'Test', email: 'test@example.com', subject: 'Test', message: 'Hello' })
  .then(() => console.log('Sent!'))
  .catch(e => console.error(e.message));
"
```
